# Adaptive Agent Orchestrator v0.5.1

Stable reliability release · 2026-07-20

## 中文

v0.5.1 修复的不是“创建更多 Agent”，而是确保已经创建的 Worker 不会被
误判、重复创建或丢失最终结果。

### 主要更新

- 创建前以文件系统原子操作保留 activation key；后台创建调用无论返回成功
  还是错误，都必须结合来源任务、创建时间窗和任务摘要核对任务列表。
- 创建 reservation 前必须保存角色启用说明并绑定其哈希；主 Agent 仍须在
  调用创建工具前把说明展示给用户，文件本身不冒充“用户已看见”。
- 唯一匹配会被采纳；多个匹配会选出一个确定性主实体并进入
  `duplicates_pending`，处理重复项前不继续；只有重复空快照达到可见性窗口
  末端才可判定未创建，重试还必须绑定原始输入、reservation 和回执哈希。
- `wait_threads` 只作为独立后台线程的可选等待优化。原生 subagent 使用其
  自己的列表和等待机制；后台结果以 `read_thread` 为主要回收路径。
- 必需后台节点必须保存原始 `read_thread` 捕获，并生成包含最终轮次、真实
  线程、捕获哈希、内容哈希和采纳/拒绝结论的结果回执，才能通过完成门。
- 确定性失败后不能静默重新创建 Worker；必须由用户授权该次失败事件。run
  内规范化前提清单只作为审计上下文，不能自行解锁重试。

### 不包含

- 不增加 Router Agent、角色圆桌或新的模式名称。
- 不改变 Luna/Sol 默认模型池，也不自动启用 Terra 或 Ultra。
- 不要求用户设置 Token budget。

### 验证

- 21 个 PowerShell 脚本通过语法解析；
- 464 项自测断言通过；
- 47 份故意构造的非法计划被正确拒绝；
- 新增创建错误后成功实体化、单次空列表保持未知、双次无匹配、重复线程
  去重、结果回执篡改和相同前提重复失败等反例。

## English

v0.5.1 prevents created Workers from being misclassified, duplicated, or
integrated without a collected final result.

### Highlights

- Atomically reserve each activation key, then reconcile every background
  creation call against source thread, creation window, and task summary,
  regardless of the call's returned status.
- Bind each durable reservation to a saved role-activation preview and its
  hash, while still requiring the main agent to show that explanation before
  invoking the creation tool.
- Adopt a unique match, choose one deterministic canonical thread and stop in
  `duplicates_pending` when duplicates exist, and keep a single empty snapshot
  in `unknown`. A no-match retry must reach the visibility-window end and bind
  the raw reconciliation input, activation reservation, and receipt hashes.
- Treat `wait_threads` as optional for independent background threads. Native
  subagents use `list_agents` and `wait_agent`; independent background agents
  use `list_threads` and `read_thread`.
- Require a hash-bound final-result receipt tied to a captured `read_thread`
  response before a required background node can pass completion.
- Require user authorization bound to the exact prior event after every
  deterministic failure. A canonical run-local premise manifest remains audit
  context and cannot authorize its own retry.

### Deliberately unchanged

- No Router Agent, role roundtable, or additional mode names.
- No automatic Terra or Ultra routing.
- No user Token budget.

### Validation

- All 21 PowerShell scripts pass parser validation.
- 464 self-test assertions pass.
- 47 intentionally invalid plans are rejected.
- New cases cover ambiguous creation, delayed materialization, duplicate
  threads, result-receipt tampering, and repeated deterministic failure.
