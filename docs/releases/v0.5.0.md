# Adaptive Agent Orchestrator v0.5.0

Release date: 2026-07-19
Channel: stable

## 中文

v0.5.0 是第一个不带预发布后缀的正式版本。它没有重新设计整个 Skill，而是
把前几个版本已经验证的角色、上下文和恢复机制，与本版新增的模式、模型和
容量规则整合成稳定发布。

### 主要更新

- `auto` 确定性解析为 `quick`、`team` 或 `workflow`，不创建额外调度
  Agent。
- `lean / balanced / quality` 只控制验证强度，不通过增加角色或 Worker
  冒充更高质量。
- 默认自动模型池只包含 Luna 和 Sol；Terra 仅在用户明确指定时作为实验
  模型使用。
- 模型或 reasoning effort 升级前必须取得用户确认；Ultra 必须逐节点
  确认。
- 目标容量为四个活动常驻 Worker 加两个受保护的临时 subagent 槽位，并
  根据实际运行时容量自动缩减。
- 新增按需研究证据角色。只有多个下游任务需要复用同一来源集，或用户明确
  要求建立证据库时才启用。
- 计划、创建预览、任务包和事件日志同时记录计划模型与实际模型，禁止静默
  替换。
- 重试升级从通过完整性校验的上一轮日志读取实际模型，不能由调用方自行
  填写。

### 相对 v0.4.2 的变化

v0.4.2 解决的是“启用什么角色、为什么创建 Worker”；v0.5.0 进一步解决：

- 应采用轻量直达、独立团队还是可恢复工作流；
- 哪些任务可以使用较轻模型，哪些任务需要 Sol；
- 如何在保留常驻角色的同时为临时 subagent 留出执行空间；
- 何时值得建立可复用研究证据，而不是让主 Agent 和多个 Worker 重复检索。

角色仍然不等于 Worker，主 Agent 仍是唯一调度者。用户不需要填写 Token
budget，也不需要操作模式与质量档位的组合表。

### 升级说明

- 持久计划的 `policy_version` 从 `0.4.2` 升级到 `0.5.0`。
- 旧计划不会被静默改写；需要继续运行时，应明确迁移并重新验证。
- Terra 不再是自动候选，也不会作为默认回退。
- 六个 Worker 是目标上限，不是平台承诺；运行时容量更低时必须缩减。

### 验证

- 18 个 PowerShell 脚本解析通过；
- 438 项自测断言通过；
- 47 个故意构造的非法计划正确拦截；
- Skill Creator 校验通过；

## English

v0.5.0 is the first stable-channel release without a prerelease suffix. It
does not replace the project wholesale. It consolidates the role, context, and
recovery controls validated in earlier versions with deterministic mode,
model, and active-capacity rules.

### Highlights

- Resolve `auto` deterministically to `quick`, `team`, or `workflow` without a
  separate routing Agent.
- Keep `lean / balanced / quality` limited to verification intensity; higher
  quality never means filling more seats by default.
- Use Luna and Sol in the automatic model pool. Terra remains an explicit
  user-selected experiment.
- Require confirmation before model or reasoning-effort escalation. Ultra
  requires confirmation for each node.
- Target four active persistent Workers plus two protected transient-subagent
  slots, clamped to actual runtime capacity.
- Activate a research-evidence role only when multiple downstream tasks will
  reuse the same source set or the user explicitly requests an evidence base.
- Record both planned and actual model identity in previews, packets, and
  durable events.
- Derive retry escalation from the integrity-checked prior journal instead of
  trusting caller-supplied prior-model claims.

### Evolution from v0.4.2

v0.4.2 made role activation visible and reviewable. v0.5.0 additionally
decides the execution shape, selects the bounded model class, protects
transient capacity beside persistent roles, and prevents repeated research
across downstream workstreams.

Roles still do not imply Workers, and the main agent remains the sole
orchestrator. Users do not configure a Token budget or a mode/profile matrix.

### Upgrade notes

- Durable plans move from policy version `0.4.2` to `0.5.0`.
- Existing plans are never rewritten silently; migrate and revalidate them
  before resuming.
- Terra is neither an automatic candidate nor a default fallback.
- Six active Workers is a target ceiling, not a platform guarantee.

### Validation

- 18 PowerShell scripts parsed;
- 438 self-test assertions passed;
- 47 intentionally invalid plans were correctly rejected;
- Skill Creator validation passed;
- the independent read-only architecture review returned `GREEN` with no
  remaining P0/P1 findings.
